# KB차차차 통합 파이프라인 설계

## 목표
KB차차차 크롤링 → 워터마크 제거/로고 교체 → 정적 홈페이지 반영을 하나의 파이프라인으로 통합

## 모듈 구조

```
kb/
├── config.py         # 공통 설정
├── crawler.py        # 크롤링 + 데이터 정제
├── watermark.py      # 워터마크 제거 + SH GLOBAL 로고
├── homepage.py       # 정적 사이트 갱신
├── pipeline.py       # 오케스트레이션
├── state.json        # 크롤링 상태
├── cars.json         # 정제된 차량 데이터
└── data/             # 차량별 폴더 (차량명/photo_N.jpg)
```

## 모듈별 역할

### config.py
- 경로 설정 (상대경로 기반)
- 브랜드/연료/색상/변속기 한→영 매핑 테이블
- 워터마크 좌표, 로고 설정
- 환율 설정 (기본값 + 설정 파일 분리)

### crawler.py (기존 kb_crawler.py 개선)
- Playwright 비동기 크롤링
- **데이터 정제 개선:**
  - 차량명에서 `· 가솔린 · 경기 매물번호(...)` 제거 → 순수 모델명만
  - `가격` 필드에서 번호판 제거, 차량명 재파싱
  - 불필요 필드 제거 (내차사기, 내차팔기, 혜택, 안내 등)
  - 옵션 노이즈 필터 강화 (대출, 중고차대출 등 제외)
- carSeq 기반 중복 방지
- 카매니저 중복 체크 로직 제거
- 저장: `kb/data/{차량명}/` 하위에 info.json + photo_N.jpg

### watermark.py (기존 kb_watermark.py 유지)
- Seamless Clone 방식 워터마크 제거
- 워터마크 위치: 우측 상단 (y: 4.5%~12%, x: 80%~98%)
- SH GLOBAL 로고 배치 (logo_clean.png, LOGO_HEIGHT_RATIO: 0.09)
- 처리 완료 폴더 추적 (신규만 처리)

### homepage.py (기존 update_homepage.py 개선)
- KB 데이터 전용으로 변경
- `kb/data/` 폴더에서 차량 정보 읽기
- 정적 HTML에 carsData JS 배열 갱신
- carSeq 기반 중복 업로드 방지 (deployed 목록 관리)
- 브랜드/연료 필터 자동 갱신

### pipeline.py
- 순차 실행: crawler → watermark → homepage
- 각 단계 성공/실패 로깅
- 개별 단계만 실행 가능 (CLI 인자)

## 데이터 정제 규칙

차량명 정제:
- 입력: `"롤스로이스 컬리넌 · 가솔린 · 경기   매물번호(28008008)"`
- 또는: `"(175서1789)롤스로이스 컬리넌6.7 V12 Black Badge"` (가격 필드)
- 출력: `"롤스로이스 컬리넌 6.7 V12 Black Badge"`

필터링할 필드:
- `내차사기`, `내차팔기`, `혜택`, `안내`, `외부패널`, `프레임`, `전손이력`(값이 정상/없음일때), `침수이력`(값이 없음일때), `엔진`(보증 설명), `미션`(보증 설명), `일반`(보증 설명)

유지할 필드:
- 차량명, carSeq, 가격(만원), 연식, 주행거리, 연료, 변속기, 배기량, 차종, 색상, 옵션, 사진URLs, 전손이력(있음일때), 침수이력(있음일때), 소유자변경

## 중복 방지
- `state.json`에 `collected_car_seqs` 관리 (크롤링 단계)
- `deployed_car_seqs` 관리 (홈페이지 반영 단계)
- 신규 차량만 추가, 기존 차량 건드리지 않음 (A 방식)
